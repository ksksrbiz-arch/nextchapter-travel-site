## Best Practices for Maintaining These Files

### Updating Guidelines

1. **Keep synchronized**: Both files should reflect the same technical reality
2. **Update together**: When project changes, update both files
3. **Maintain consistency**: Keep architecture descriptions aligned
4. **Version history**: Add entries to version history when making changes
5. **Test instructions**: Verify that instructions still work

### When to Update

Update both files when:
- ✅ New pages or components are added
- ✅ Security requirements change
- ✅ Coding standards evolve
- ✅ New testing tools are adopted
- ✅ Browser support matrix changes
- ✅ Accessibility requirements update
- ✅ New troubleshooting patterns emerge

### What NOT to Include

Do not include in these files:
- ❌ Temporary notes or TODOs
- ❌ Personal preferences
- ❌ Experimental features
- ❌ Deprecated practices
- ❌ Internal team communications
- ❌ Sensitive information (credentials, API keys)
